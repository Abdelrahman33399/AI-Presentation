معايير بناء الموقع (Front-End & UX)

التصميم والتوافق

  • تصميم متجاوب بالكامل (Responsive Design) — يعمل على الجوال، التابلت، والديسكتوب
  • مبدأ Mobile-First — تطوير بدءًا من الشاشات الصغيرة
  • دعم كامل لاتجاه RTL (اللغة العربية من اليمين لليسار)
  • متوافق مع المتصفحات الرئيسية (Chrome, Firefox, Edge, Safari)

إمكانية الوصول (Accessibility)

  • متوافق مع معايير WCAG 2.1 (AA Level)
  • دعم كامل لقارئات الشاشة (Screen Readers) عبر semantic HTML + ARIA
  • تباين ألوان عالي بين النص والخلفية (≥ 4.5:1)
  • وضع مظلم/فاتح قابل للتخصيص وحفظه عبر localStorage

الأداء والتجربة

  • سرعة تحميل عالية (Lighthouse Performance ≥ 90)
  • تحميل كسول (Lazy Loading) للصور والفيديوهات
  • تحسين حجم الصور (WebP + q=80)
  • تأثيرات أنيميشن سلسة بدون تأثير على الأداء

التقنيات المستخدمة

  • HTML5 + CSS3 (Grid, Flexbox, Custom Properties)
  • Vanilla JavaScript (بدون مكتبات خارجية — لتقليل الحجم)
  • Canvas API للخلفيات التفاعلية
  • Intersection Observer لـ Scroll-based Animations

معايير العرض التقديمي (Presentation & EdTech Design)

التخطيط التعليمي

  • هيكلة المحتوى وفق نموذج Gagne’s Nine Events of Instruction
  • دمج أسئلة تفاعلية بعد كل وحدة لتثبيت التعلّم
  • تخصيص المحتوى حسب أنماط التعلّم (بصري، سمعي، حركي)
  • دعم Universal Design for Learning (UDL)

التصميم البصري (Presentation Design)

  • الالتزام بمبدأ 6×6 (لا أكثر من 6 أسطر، 6 كلمات/سطر)
  • استخدام خطوط واضحة (Cairo Arabic) بحجم ≥ 24pt للعناوين
  • ألوان متناسقة مع هوية بصرية ثابتة (Neon Green + Orange)
  • صور ذات جودة عالية، مع وصف نصي بديل (Alt Text)

التفاعل والدمج

  • كروت تفاعلية مع تأثيرات Hover (Zoom, Glow, Lift)
  • أسئلة اختيار من متعدد مع تغذية راجعة فورية + شرح
  • دعم إعادة الاختبار وتقييم الأداء
  • ربط المحتوى بالتطبيقات الواقعية (مثل: Seeing AI, Proloquo2Go)

الجودة والاحترافية

  • مراجعة لغوية دقيقة (بدون أخطاء إملائية أو نحوية)
  • توثيق علمي للمصادر (عند الحاجة)
  • استخدام لغة واضحة، بسيطة، ودقيقة — مناسبة لطلاب الجامعات
  • صور واقعية/كرتونية مُصممة خصيصًا لتعكس الهوية الثقافية
